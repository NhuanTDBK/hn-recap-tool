PostgreSQL 18 Released
HN Discussion
PostgreSQL 18 brings significant performance improvements to OLTP workloads with up to 2x throughput on high-connection counts. Key additions include native JSON path indexing, async I/O for vacuum, and improved logical replication. The release also features better query optimization and enhanced monitoring capabilities for production environments. The async I/O for vacuum is particularly significant as it addresses a long-standing bottleneck in maintaining large tables. Users can now perform maintenance without blocking concurrent operations. JSON path indexing brings PostgreSQL closer to modern document-oriented databases while maintaining ACID guarantees. Performance benchmarks show the improvements are consistent across different hardware configurations. The new logical replication enhancements make it easier to build distributed PostgreSQL clusters.
